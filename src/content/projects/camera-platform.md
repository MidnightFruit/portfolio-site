## Задача

Нужна была основа для работы с промышленными камерами, на которой можно строить разные
измерительные приложения. Главные требования: поддержка камер от разных производителей,
синхронный захват кадров с нескольких камер одновременно и возможность работать как с GUI,
так и headless на сервере.

## Моя роль

Спроектировал архитектуру платформы и реализовал её как программу сбора данных.
Решения по структуре - разделение на ядро, интерфейсы и прокси-слой, выбор процессов
для синхронного захвата - мои. Параллельно команда работала над вычислительным решателем,
который использует данные с платформы.

Изначально это была отдельная программа управления камерами. Со временем она стала основой,
поверх которой построены остальные проекты - анализ вибраций, измерение деформаций и другие.

## Архитектурные решения

**Прокси-слой через единый интерфейс `ICam`.**
Чтобы добавить камеру нового производителя, достаточно реализовать методы интерфейса `ICam` -
ядро при этом не меняется. Сейчас через этот механизм интегрированы web-камеры (стандартный
драйвер OpenCV) и промышленные камеры Daheng (общий вендорский SDK). Новый SDK подключается
по той же схеме.

**Процессы вместо потоков для захвата.**
Одно из требований - синхронные снимки с нескольких камер с минимальной задержкой. Потоки
здесь не подходили: из-за GIL они не дают настоящего параллелизма, и задержка получалась
слишком большой. Планировщик по времени тоже был недостаточно быстрым - кадры нужно было
получать как можно быстрее. Решение - отдельный процесс на каждую камеру, что убирает
конкуренцию за GIL и даёт реальную параллельность.

**Разделение ядра и интерфейсов.**
Ядро (Command Controller + управление камерами) не зависит от способа доступа. Поверх него
работают GUI и External API - за счёт этого платформу можно запускать как настольное
приложение или headless на сервере.

## Текущее состояние

- Ядро, GUI и работа с несколькими камерами - реализованы
- External API (REST/WebSocket/gRPC) - реализованы демонстрационные методы (включение камеры,
  получение кадра); задуман для headless-запуска на сервере
- CLI - запланирован
- Интегрированы: OpenCV-камеры, SDK Daheng
